# 포트 관리기 사용자 가이드

> 처음 쓰는 분도 5분이면 핵심 기능을 모두 쓸 수 있습니다.

---

## 이 앱이 뭔가요?

**포트 관리기**는 세 가지를 한 곳에서 해결합니다.

| 화면 | 하는 일 | 접근 방법 |
|---|---|---|
| **프로젝트 관리** | `.command` 파일로 개발 서버를 한 클릭으로 실행/중지/재시작 | 로컬 앱 (데스크톱 전용) |
| **포털** | 자주 쓰는 사이트·폴더를 카테고리별 북마크로 관리 | 로컬 앱 + 모바일 |
| **북마크 웹버전** | 포털과 동일한 북마크 기능 — 어디서나 브라우저로 접속 | Vercel 배포 URL |

Supabase 연결 시 **세 화면 모두 자동 동기화**됩니다.

---

## 5분 퀵스타트

### Step 1. 접속
- 브라우저: `http://localhost:9000`
- 데스크톱 앱: `포트관리기.app` 실행

[스크린샷: 앱 첫 화면 — 상단 탭 "프로젝트 관리" / "포털" 확인]

### Step 2. 첫 포트 등록 (가장 쉬운 방법 — 드래그 앤 드롭)
1. Finder에서 `.command` 파일을 찾습니다
2. 앱 창으로 **끌어다 놓기**
3. 포트 번호와 프로젝트 폴더가 **자동 감지**됩니다

[스크린샷: .command 파일을 드래그하면 자동 등록되는 장면]

> `.command` 파일이 없다면? **⚙ 세팅** → 초기 설정 마법사

### Step 3. 서버 실행
프로젝트 카드의 **▶ 실행** 버튼 클릭
→ 우측 상단 초록색 알림 Toast
→ 카드 상태가 **실행 중** (초록)으로 변경

[스크린샷: 실행 중 상태 카드 — 초록 배지 + Toast 알림]

### Step 4. 브라우저에서 열기
카드의 **🌐 열기** 버튼 → `localhost:포트`가 Chrome에서 자동 오픈

**여기까지 완료 = 핵심 워크플로우 성공!**

## 프로젝트 관리 탭 완전 정복

[스크린샷: 프로젝트 관리 탭 전체 화면 — 툴바, 검색, 카드 목록]

### 화면 구성 (위 → 아래)

```
[내보내기] [불러오기] [새로고침] [Push ↑] [Pull ↓] [⚙ 세팅] [가이드]
─────────────────────────────────────────────────────────────────────
[🔍 프로젝트 검색... (Cmd+F)]
[전체] [포트 있음] [포트 없음]                      [정렬 옵션]
─────────────────────────────────────────────────────────────────────
  프로젝트 카드 목록
```

### 프로젝트 카드 버튼 설명

[스크린샷: 프로젝트 카드 확대 — 각 버튼 위치 표시]

| 버튼 | 동작 |
|---|---|
| **▶ 실행** | `.command` 파일 실행, 로그 파일에 자동 저장 |
| **⏹ 중지** | SIGTERM → (응답 없으면) SIGKILL 자동 전환 |
| **⚠ 강제 재실행** | 모든 PID 즉시 종료 → 500ms 대기 → 재실행 |
| **🌐 열기** | Chrome에서 `localhost:포트` 오픈 |
| **📁 폴더** | Finder에서 프로젝트 폴더 열기 |
| **📋 로그** | Terminal에서 `tail -f`로 실시간 로그 확인 |
| **✏️ 수정** | 포트·이름·경로 편집 |
| **🗑 삭제** | 목록에서 제거 (파일은 삭제 안 함) |

### 검색 & 필터

- **Cmd+F**: 검색창 즉시 포커스 (이름, 경로, AI 별칭 통합 검색)
- **포트 있음**: 서버 포트가 있는 프로젝트만 표시
- **포트 없음**: `.command`만 있고 포트 없는 스크립트만 표시
- **카테고리 필터**: AI가 자동 분류한 카테고리별 보기
- **정렬**: 최근 등록순 / 이름순 / 포트순 + 오름/내림 토글

### AI 이름 자동 생성
새로고침 버튼 옆 ✨ AI 버튼 클릭
→ 폴더 내용 분석 → 프로젝트 이름·카테고리 자동 제안
→ Claude Code 설치 필요 (`npm install -g @anthropic-ai/claude-code`)

[스크린샷: ✨ AI 버튼 클릭 후 자동 분류된 카드 목록]

## 포털 탭 완전 정복

[스크린샷: 포털 탭 — 카테고리 사이드바 + 아이템 그리드]

### 화면 구성

```
[카테고리 사이드바]  |  [검색창]
                    |  [아이템 그리드]
                    |  ─ 핀 고정 항목 (최상단)
                    |  ─ 일반 항목
                    |  [+ 추가 버튼]
```

### 북마크 추가
우측 하단 **+** 버튼 → URL 또는 폴더 경로 입력

[스크린샷: 북마크 추가 모달 — URL 입력 후 자동 제목 추출]

| 타입 | 예시 | 클릭 시 |
|---|---|---|
| URL | `https://notion.so/...` | 새 탭에서 열기 |
| 폴더 | `/Users/me/Projects/` | Finder에서 열기 |

### 카드 기능 (hover 시 표시)

| 아이콘 | 기능 |
|---|---|
| 📌 핀 | 상단 고정 목록에 표시 |
| ✏️ 연필 | 북마크 편집 |
| 🗑 휴지통 | 삭제 |

방문 횟수 & 마지막 방문 시각이 자동으로 기록됩니다.

### 카테고리 관리
좌측 사이드바 **+** → 이름 + 색상 선택
카테고리는 **전 기기 공유** (어느 기기에서 만들어도 모든 기기에 보임)

### 기기별 격리 정책

| 항목 | 공유 범위 |
|---|---|
| URL 북마크 | 전 기기 공유 |
| 카테고리 | 전 기기 공유 |
| 폴더 북마크 | 기기별 개별 (경로가 달라서) |

## 북마크 웹버전 (어디서나 접속)

**포털 탭과 동일한 북마크 기능**을 Vercel에 배포된 독립 웹앱으로 제공합니다.
회사 PC, 모바일, 다른 사람 컴퓨터 어디서든 URL 하나로 내 북마크에 접근할 수 있습니다.

[스크린샷: 북마크 웹버전 — 비밀번호 입력 화면]

### 첫 접속 흐름

1. **비밀번호 입력** (설정된 경우) → "이 브라우저에서 기억하기" 체크 시 다음부터 스킵
2. **기기 선택** → Supabase에 등록된 기기 목록에서 내 기기 선택
   - 목록에 없으면: Device ID 직접 입력 또는 새 기기로 등록
3. **자동 Pull** → 선택한 기기의 북마크 데이터 즉시 불러오기

[스크린샷: 기기 선택 화면 — 등록된 기기 목록에서 클릭]

### 주요 특징

| 특징 | 설명 |
|---|---|
| **어디서나 접속** | Vercel URL로 브라우저만 있으면 OK |
| **비밀번호 보호** | 배포 시 `VITE_PORTAL_PASSWORD_HASH` 환경변수로 설정 |
| **자동 동기화** | 접속 시 Push/Pull로 Supabase와 즉시 동기화 |
| **모바일 최적화** | 반응형 레이아웃 — 스마트폰에서도 편하게 사용 |
| **포털 탭과 동일** | 카테고리·핀·방문 횟수 등 모든 기능 동일 |

[스크린샷: 북마크 웹버전 메인 화면 — 카테고리 + 아이템 그리드 (모바일)]

### 포털 탭 vs 북마크 웹버전 비교

| | 포털 탭 (로컬) | 북마크 웹버전 |
|---|---|---|
| 접속 방법 | localhost:9000 또는 .app | Vercel URL |
| 인터넷 연결 | 불필요 | 필요 |
| 프로젝트 관리 탭 | 같이 있음 | 없음 (북마크만) |
| 폴더 북마크 | Finder 열기 가능 | 해당 기기만 가능 |
| 모바일 접속 | 불편 | 최적화 |

## 다기기 동기화 (Supabase)

[스크린샷: 설정 모달 — Supabase URL + anon key 입력]

### 첫 설정 (한 번만)
1. **⚙ 세팅** → Supabase URL + anon key 입력
2. (또는 초기 설정 마법사에서 자동 안내)

### 동기화 방법

| 버튼 | 동작 |
|---|---|
| **Push ↑** | 이 기기 데이터 → Supabase 업로드 |
| **Pull ↓** | Supabase → 이 기기 복원 |

### 다른 기기 데이터 가져오기
**⚙ 세팅** → 고급 설정 → **단말 조회** → 기기 선택 → **저장 후 동기화**
→ 해당 기기의 포트 목록을 Pull

## 자주 겪는 문제 해결

### 서버가 안 켜져요
→ 카드의 **⚠ 강제 재실행** 클릭 (포트 점유 중인 좀비 프로세스 자동 처리)

### Push/Pull 버튼이 안 눌려요
→ **⚙ 세팅**에서 Supabase URL과 anon key가 입력되어 있는지 확인

### 포트 번호가 자동으로 안 잡혀요
→ `.command` 파일 안에 `PORT=3000` 또는 `localhost:3000` 형식 포함 필요

### 다른 기기에서 내 폴더 경로가 비어있어요
→ 정상입니다. Pull 후 경로 설정 창에서 이 기기의 경로를 지정하세요.

### 북마크 웹버전에서 내 기기가 목록에 없어요
→ Device ID 직접 입력 탭에서 포트 관리기 설정에서 확인한 UUID를 붙여넣으세요.
