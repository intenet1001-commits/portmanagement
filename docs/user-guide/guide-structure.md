# 포트 관리기 사용자 가이드 (초안)

> **대상**: 처음 사용하는 초보자 / 무슨 기능이 있는지 모르는 사용자
> **목표**: 앱 접속 → 5분 안에 첫 성공 경험 → 주요 기능 자연스럽게 발견

---

## 0. 이 앱이 뭔가요? (30초 소개)

**포트 관리기**는 두 가지를 한 곳에서 관리하는 도구입니다.

1. **프로젝트 관리 탭**: 내 맥/윈도우의 `.command` 파일들을 모아두고, 한 번 클릭으로 로컬 개발 서버(예: `localhost:3000`)를 띄우고/끄는 런처
2. **포털 탭**: 자주 쓰는 사이트/폴더/도구를 카테고리별로 정리하는 개인 북마크 + 런처

**특징**: 여러 기기(집/회사 맥)에서 같은 목록을 동기화할 수 있음 (Supabase).

---

## 1. 5분 퀵스타트

### Step 1 — 접속 (10초)
브라우저에서 `http://localhost:9000` 접속 또는 데스크톱 앱(`포트관리기.app`) 실행.

### Step 2 — 첫 화면 둘러보기 (30초)
상단 탭이 두 개 보입니다.
- **프로젝트 관리** — 개발 서버 목록 (데스크톱)
- **포털** — 북마크 (모든 기기)

모바일/태블릿에서는 **포털만** 보입니다.

### Step 3 — 첫 포트 등록 (2분)
> 가장 쉬운 방법: **드래그 앤 드롭**

1. Finder에서 `.command` 파일을 찾습니다 (예: `내프로젝트-실행.command`).
2. 앱 창으로 파일을 **그대로 끌어다 놓습니다**.
3. 끝! 포트 번호와 프로젝트 폴더가 자동으로 감지됩니다.

> `.command` 파일이 없다면? 상단 **세팅** 버튼 → 설정 마법사 → 가이드 따라하기

### Step 4 — 서버 실행 (30초)
등록된 프로젝트 카드에서 **▶ 실행** 버튼 클릭.
- 우측 상단에 초록색 Toast ("실행되었습니다")
- 카드 상태가 **실행 중** (초록색)으로 바뀜

### Step 5 — 브라우저에서 열기 (20초)
**Chrome에서 열기** 버튼 클릭 → 자동으로 `localhost:포트`가 열립니다.

> 여기까지 하셨다면 **핵심 워크플로우 완료**입니다.

## 2. 주요 사용 시나리오 5가지

### 시나리오 A — "매일 아침 5개 프로젝트 일괄 실행"
**문제**: 매일 터미널 5개 열고 각자 `bun run dev` 입력하기가 번거로움.
**해결 흐름**:
1. 프로젝트마다 `.command` 등록 (한 번만)
2. 아침에 앱 열고 **▶ 실행**을 5번 클릭
3. **Chrome에서 열기**로 한 번에 브라우저 탭 오픈

### 시나리오 B — "포트가 꼬여서 서버가 안 뜸"
**문제**: 좀비 프로세스가 포트 3000을 점유 중이라 서버 실행 실패.
**해결 흐름**:
1. 카드에서 **⚠ 강제 재실행** 버튼 클릭
2. 내부적으로 `SIGKILL`로 모든 PID 종료 → 500ms 대기 → 재실행
3. Toast로 결과 확인

### 시나리오 C — "집 맥 설정을 회사 맥에 옮기기"
**문제**: 집에서 등록한 프로젝트 20개를 회사 맥에 다시 등록하기 싫음.
**해결 흐름**:
1. 집 맥: 상단 **세팅** → Supabase URL + anon key 입력 (1회)
2. 집 맥: **Push** 버튼 클릭 (클라우드 업로드)
3. 회사 맥: 같은 Supabase 설정 → **Pull** 버튼
4. **경로 remapping 모달** 자동 표시 → 회사 맥 폴더 경로 한 번에 지정

### 시나리오 D — "실행 로그를 보고 싶다"
**문제**: 서버가 뭘 하는지, 에러가 나는지 안 보임.
**해결 흐름**:
1. 카드의 **로그** 버튼 클릭
2. Terminal 창이 자동으로 열리고 `tail -f`로 실시간 출력 표시

### 시나리오 E — "북마크를 탭 대신 여기서"
**문제**: 자주 쓰는 Notion / Figma / GitHub 북마크를 브라우저가 아닌 전용 런처에서 관리하고 싶음.
**해결 흐름**:
1. **포털** 탭 이동
2. 카테고리 생성 (예: "디자인", "회사", "개인")
3. URL 붙여넣기 → 자동 제목 추출
4. 📌 핀 고정 / 방문 횟수 자동 트래킹

## 3. "이런 기능이 있었어?" 섹션 (숨겨진 보석)

초보자가 사용하면서 "헉, 이런 것도 돼?" 하는 기능들.

| 기능 | 어디서 | 한마디 설명 |
|---|---|---|
| **Cmd+F 검색** | 프로젝트 관리 탭 | 프로젝트 검색창 인스턴트 포커스 |
| **AI 이름/카테고리 자동 생성** | 새로고침 옆 AI 버튼 | `claude -p`로 폴더 내용 분석 → 예쁜 이름 제안 |
| **포트 필터 (전체/있음/없음)** | 검색창 아래 탭 | `.command`만 있고 포트 없는 스크립트 구분 |
| **DMG 출시하기** | 세팅 → 빌드 메뉴 | 빌드된 DMG를 Desktop으로 자동 복사 |
| **로그 복사 버튼** | 최상단 툴바 | 앱 오류 로그 한 번에 클립보드 복사 (문의용) |
| **bypass permissions 토글** | 프로젝트 관리 탭 우상단 | `claude --dangerously-skip-permissions` 모드 전환 |
| **기기별 격리 + 공유** | 포털 탭 | 폴더는 기기별, 링크/카테고리는 전 기기 공유 |
| **단말 조회로 다른 맥 데이터 Pull** | 세팅 → 고급 설정 | 드롭다운에서 기기 선택 후 저장하면 해당 기기 데이터 가져옴 |
| **JSON 내보내기/불러오기** | 툴바 ⬇ ⬆ 아이콘 | Supabase 없이도 백업/복원 가능 |
| **실시간 빌드 로그 모달** | 세팅 → 빌드 | 앱/DMG 빌드 중 터미널 스타일로 진행 상황 표시 |

## 4. 탭별 기능 설명 흐름

### 4-1. 프로젝트 관리 탭 (데스크톱 전용)

**목적**: 로컬 개발 서버 런처 + 관리자

**구성 (위 → 아래)**:
1. **상단 툴바** — 내보내기 / 불러오기 / 새로고침 / Push / Pull / 세팅
2. **검색창** (`Cmd+F`) — 프로젝트 이름/폴더 경로 인스턴트 검색
3. **필터 탭** — 전체 / 포트 있음 / 포트 없음
4. **카테고리 필터** — AI가 분류한 카테고리별 집합 (웹앱/데스크톱/MCP 등)
5. **정렬** — 최근 등록순 / 이름순 / 포트순 + 오름/내림
6. **bypass permissions** 토글 — Claude 권한 우회 실행 여부
7. **프로젝트 카드 리스트** — 각 카드마다:
   - 이름, 포트, 폴더 경로, 실행 상태
   - 버튼: 실행 / 중지 / 강제 재실행 / Chrome에서 열기 / 폴더 열기 / 로그 / 수정 / 삭제

**초보자 학습 순서 제안**:
1. 드래그 앤 드롭으로 1개 등록 → 실행 → Chrome 열기 (성공 경험)
2. 로그 버튼 눌러보기 (서버가 살아 있다는 확인)
3. 중지 / 강제 재실행 차이 체험
4. AI 이름 보강 실행 → 자동 분류 경험
5. Push 한 번 눌러서 클라우드 저장

### 4-2. 포털 탭 (모든 기기)

**목적**: 개인 북마크 + 자주 쓰는 폴더 빠른 접근

**구성**:
1. **카테고리 사이드바** (색상 커스터마이즈 가능)
2. **아이템 그리드**:
   - `url` 타입: 웹사이트 (제목 자동 추출)
   - `folder` 타입: 로컬 폴더 (Finder에서 열기)
   - `path` 타입: 파일/커맨드
3. **아이템 메타**: 핀 여부 / 방문 횟수 / 마지막 방문 시각
4. **검색** + 카테고리 필터

**격리 정책 (중요)**:
- URL/파일 북마크 + 카테고리 → **전 기기 공유** (`device_id = '__shared__'`)
- 폴더 북마크 → **기기별 개별** (경로가 달라서)

**초보자 학습 순서 제안**:
1. 카테고리 1~2개 만들기
2. 자주 가는 사이트 URL 붙여넣기
3. 핀 고정 시도
4. 여러 기기에서 Pull해서 공유 확인

## 5. 공통 관리 기능

### 5-1. 세팅 버튼 (최상단 툴바)
- **초기 설정 마법사** — 처음 접속 시 자동 오픈. Supabase 연결부터 첫 `.command` 등록까지 안내
- **포털/Supabase 설정 모달**에서 가능한 것:
  - Supabase URL + anon key 입력
  - 기기 이름 편집 (예: "집 맥북" / "회사 맥미니")
  - 고급 설정: 단말 조회 → 다른 기기 데이터 가져오기
  - SetupGuide: "Claude 프롬프트 복사" 버튼으로 DDL 스크립트 클립보드 복사

### 5-2. 로그 복사 버튼
문제 발생 시 `logs` 복사 → 개발자에게 전달 → 빠른 진단.

### 5-3. Toast 알림 시스템
오른쪽 상단에서 슬라이드 인. 성공=초록 / 실패=빨강. 3초 후 자동 사라짐.

## 작성 가이드라인

- 각 섹션 **80자 이하 짧은 문장** 유지
- 모든 버튼은 **실제 UI 텍스트** 그대로 표기 (예: "▶ 실행", "⚠ 강제 재실행")
- 스크린샷 슬롯은 `[스크린샷: 설명]` 형식으로 TODO 마킹
- 초보자가 한 번에 **하나씩만** 실행하도록 단계 분리
- 실패 케이스는 반드시 "이런 Toast가 뜨면 ~~ 하세요" 형식으로 안내
